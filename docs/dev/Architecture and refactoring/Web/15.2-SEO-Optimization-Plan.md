## 二、优化方案（按优先级排序）

### P0：基础搜索引擎可见性

#### 2.1 创建 sitemap.ts

**目标**：让搜索引擎发现所有公开页面（静态 + 动态）

**文件**：`src/app/sitemap.ts`

```typescript
import { MetadataRoute } from "next";
import { createClient } from "@/lib/supabase/server";

const BASE_URL = "https://researchopia.com";

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  // 静态页面
  const staticPages: MetadataRoute.Sitemap = [
    { url: BASE_URL, lastModified: new Date(), changeFrequency: "daily", priority: 1.0 },
    { url: `${BASE_URL}/papers`, lastModified: new Date(), changeFrequency: "daily", priority: 0.9 },
    { url: `${BASE_URL}/webpages`, lastModified: new Date(), changeFrequency: "daily", priority: 0.8 },
    { url: `${BASE_URL}/search`, changeFrequency: "weekly", priority: 0.6 },
    { url: `${BASE_URL}/feed`, changeFrequency: "hourly", priority: 0.7 },
  ];

  // Guide 页面（静态 MDX）
  const guidePages = getGuideRoutes().map((route) => ({
    url: `${BASE_URL}/guide/${route}`,
    changeFrequency: "monthly" as const,
    priority: 0.7,
  }));

  // 动态页面：论文详情（从数据库查询）
  const supabase = await createClient();
  const { data: papers } = await supabase
    .from("papers")
    .select("doi, updated_at")
    .order("updated_at", { ascending: false })
    .limit(5000);

  const paperPages: MetadataRoute.Sitemap = (papers || []).map((p) => ({
    url: `${BASE_URL}/papers/${encodeURIComponent(p.doi)}`,
    lastModified: p.updated_at ? new Date(p.updated_at) : undefined,
    changeFrequency: "weekly",
    priority: 0.6,
  }));

  // 动态页面：用户主页
  const { data: profiles } = await supabase
    .from("profiles")
    .select("username, updated_at")
    .not("username", "is", null)
    .limit(5000);

  const profilePages: MetadataRoute.Sitemap = (profiles || []).map((p) => ({
    url: `${BASE_URL}/profile/${p.username}`,
    lastModified: p.updated_at ? new Date(p.updated_at) : undefined,
    changeFrequency: "weekly",
    priority: 0.5,
  }));

  return [...staticPages, ...guidePages, ...paperPages, ...profilePages];
}

// 辅助函数：获取所有 Guide 路由
function getGuideRoutes(): string[] {
  // 根据实际 guide 内容结构填充
  return [
    "getting-started/overview",
    "getting-started/quick-start",
    "website/paper-evaluation",
    "zotero-plugin/installation",
    "browser-extension/installation",
    "best-practices/workflow",
  ];
}
```

**要点**：
- 使用 Next.js 内置 `MetadataRoute.Sitemap` 类型，自动生成 `/sitemap.xml`
- 包含静态页、Guide 页、论文详情页、用户主页
- 通过 Supabase 查询动态内容的 DOI / username
- 如果页面超过 50,000 条，需拆分为多个 sitemap（使用 `generateSitemaps()`）

#### 2.2 创建 robots.ts

**文件**：`src/app/robots.ts`

```typescript
import { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      {
        userAgent: "*",
        allow: "/",
        disallow: [
          "/api/",
          "/auth/",
          "/bookmarks",
          "/settings",
          "/_next/",
          "/admin/",
        ],
      },
    ],
    sitemap: "https://researchopia.com/sitemap.xml",
  };
}
```

**要点**：
- 屏蔽 `/api/`、`/auth/`、`/bookmarks`（登录后内容）、`/settings`、`/_next/`
- 声明 sitemap URL，引导爬虫使用 sitemap

### P1：动态页面 Metadata

#### 2.4 论文详情页 generateMetadata

**思路**：将论文详情页从纯客户端组件改为支持服务端 Metadata 生成。

**方案 A（推荐）**：拆分为 Server Component 外壳 + Client Component 内容

```typescript
// src/app/papers/[doi]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({
  params,
}: {
  params: Promise<{ doi: string }>;
}): Promise<Metadata> {
  const { doi } = await params;
  const decodedDoi = decodeURIComponent(doi);
  // 从 Supabase 或 Crossref API 获取论文基本信息
  const paper = await fetchPaperMetadata(decodedDoi);

  if (!paper) {
    return { title: "论文未找到 - 研学港" };
  }

  return {
    title: `${paper.title} - 研学港 Researchopia`,
    description: truncate(paper.abstract, 160),
    openGraph: {
      title: paper.title,
      description: truncate(paper.abstract, 200),
      type: "article",
      publishedTime: paper.published_date,
      authors: paper.authors?.map((a: string) => a),
    },
    alternates: {
      canonical: `/papers/${encodeURIComponent(decodedDoi)}`,
    },
  };
}
```

**方案 B（低侵入替代）**：使用 `generateStaticParams` + ISR 预生成热门论文。

#### 2.5 用户主页 generateMetadata

```typescript
// src/app/profile/[username]/page.tsx
export async function generateMetadata({
  params,
}: {
  params: Promise<{ username: string }>;
}): Promise<Metadata> {
  const { username } = await params;
  const profile = await fetchProfile(username);

  if (!profile) {
    return { title: "用户未找到 - 研学港" };
  }

  return {
    title: `${profile.display_name || username} - 研学港 Researchopia`,

      title: `${profile.display_name || username} - 研学港`,
      type: "profile",
      ...(profile.avatar_url && { images: [profile.avatar_url] }),
    },
    alternates: {
      canonical: `/profile/${username}`,
    },
  };
}
```

```typescript
export function OrganizationJsonLd() {
  const jsonLd = {
    "@context": "https://schema.org",

    url: "https://researchopia.com",
    logo: "https://researchopia.com/logo-main.svg",

  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
    />
  );
}
```

```typescript
export function BreadcrumbJsonLd({ items }: { items: Array<{ name: string; url: string }> }) {
  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "BreadcrumbList",
    itemListElement: items.map((item, i) => ({
      "@type": "ListItem",
      position: i + 1,
      name: item.name,
      item: `https://researchopia.com${item.url}`,
    })),
  };

  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
    />
  );
}
```

### P4：内容与国际化

#### 2.12 首页 SEO 内容优化

当前首页是纯客户端组件，搜索引擎难以索引动态内容。

**优化策略**：
- 将首页关键内容（平台介绍、功能列表、统计数据）抽取为 Server Component
- 确保首屏 HTML 包含核心关键词：
  - "学术评价平台"、"论文评分"、"学术交流社区"
  - "academic evaluation"、"paper rating"、"research community"
- H1 标签使用品牌主标语
- H2 标签覆盖核心功能模块

**关键词策略**（基于品牌定位）：

| 优先级 | 中文核心词 | 英文核心词 |
|--------|-----------|-----------|
| 高 | 学术评价、论文评分、学术社区 | academic evaluation, paper rating |
| 高 | 研学港、Researchopia | Researchopia |
| 中 | 知识共享、学术交流 | knowledge sharing, academic exchange |
| 中 | 网页评价、网页收藏 | webpage evaluation, bookmarks |
| 低 | Zotero 插件、浏览器扩展 | Zotero plugin, browser extension |

## 四、技术注意事项

### 4.1 Cloudflare 部署兼容性

当前使用 `open-next.config.ts` + Cloudflare Workers 部署。需注意：
- `sitemap.ts` 和 `robots.ts` 均为 Server Routes，需确认 Cloudflare 适配器（`@opennextjs/cloudflare`）正确处理
- Sitemap 数据库查询需在 Edge Runtime 中可用（Supabase REST API 支持）
- 如 sitemap 动态生成在 Edge 受限，可退化为构建时生成静态 `sitemap.xml`

### 4.2 客户端组件的 SEO 限制

大量页面使用 `"use client"` 指令，搜索引擎爬虫（尤其非 Google）可能无法执行 JavaScript。解决方案：
- **优先**：添加 `generateMetadata` 导出（不要求整个页面改为 Server Component）
- **次选**：使用 ISR/SSG 预渲染热门页面
- **兜底**：Google 的 Googlebot 支持 JS 渲染，但百度爬虫不支持

### 4.3 性能预算

添加 SEO 组件时注意：
- JSON-LD 脚本插入不影响渲染性能（仅在 HTML head 中）
- OG 图片动态生成（ImageResponse）的 Edge 计算时间需监控
- Sitemap 数据库查询加 `limit(5000)` 防止超时
- GA4 脚本使用 `@next/third-parties` 自动 defer，不阻塞首屏
