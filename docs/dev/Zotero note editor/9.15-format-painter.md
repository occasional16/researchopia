# 9.15 Format Painter（格式刷）

## Background

Word 的格式刷是经典的生产力工具——选中已格式化的文本，点击格式刷，再选中目标文本即可复制格式。Zotero 笔记编辑器（基于 ProseMirror）目前没有此功能。本方案在 Zotero 插件中完整实现 Word 风格的格式刷。

## Design

### 用户交互流程（完全复刻 Word）

| 操作 | 行为 |
|------|------|
| 选中文本 → **单击**格式刷 | 进入「一次性模式」：光标变为刷子，选中目标文本后自动应用格式并退出 |
| 选中文本 → **双击**格式刷 | 进入「锁定模式」：可连续多次刷格式，按 **Esc** 或再次点击按钮退出 |
| 无选区时点击 | 无操作（按钮 disabled） |
| 格式刷激活时按 Esc | 退出格式刷模式，恢复正常光标 |

### UI 注入位置

在 `.text-dropdown .popup` 下拉菜单底部注入（与 colorPicker 注入 `.color-dropdown .popup` 的方式一致）：

```
┌─────────────────────┐
│ Bold                │
│ Italic              │
│ Underline           │
│ Strikethrough       │
│ Superscript         │
│ Subscript           │
│ Code                │
│─────────────────────│  ← separator
│ 🖌️ Format Painter   │  ← 注入按钮
└─────────────────────┘
```

### 按钮状态

| 状态 | 视觉 |
|------|------|
| **默认** | 普通菜单项样式，与 Bold/Italic 等选项一致 |
| **已激活（一次性）** | 按钮高亮（背景色 `var(--fill-quinary)`），下拉菜单关闭后在 toolbar 中可见一个临时高亮的小指示器 |
| **已激活（锁定）** | 同上 + 按钮边框加粗/加深，表明持续模式 |
| **无选区** | 灰色不可点击 |

### 光标样式

格式刷激活时，编辑区域的光标需要变化以提示用户：

```css
.format-painter-active .ProseMirror {
  cursor: url('data:image/svg+xml,...') 2 14, crosshair;
}
```

使用内联 SVG data URI 避免额外文件加载，降级为 `crosshair`。

### 文件结构

```

```

### Step 1: 采集格式（Capture Marks）

读取当前选区上所有 inline marks：

```ts
function captureFormat(view: EditorView): Mark[] {
  const { state } = view;
  const { from, to, empty } = state.selection;
  
  if (empty) {
    // 光标无选区：使用 storedMarks 或光标位置的 marks
    return state.storedMarks || state.selection.$from.marks();
  }
  
  // 有选区：收集选区内所有 marks 的交集
  // （Word 行为：取选区第一个字符的格式）
  return state.doc.resolve(from).marksAcross(state.doc.resolve(to)) 
    || state.selection.$from.marks();
}
```

格式类型包含（Zotero note-editor ProseMirror schema 支持的全部 inline marks）：

| Mark | 说明 |
|------|------|
| `strong` | 粗体 |
| `em` | 斜体 |
| `underline` | 下划线 |
| `strike` / `strikethrough` | 删除线 |
| `code` | 行内代码 |
| `textColor` | 文字颜色（`attrs.color`） |
| `highlight` | 高亮颜色（`attrs.color`） |
| `superscript` | 上标 |
| `subscript` | 下标 |
| `link` | 超链接（**不刷**——复制链接到其他文本无意义） |

> **Note**: `link` mark 需要从采集结果中过滤掉。

### Step 2: 应用格式（Apply Marks）

```ts
function applyFormat(view: EditorView, marks: Mark[]): void {
  const { state } = view;
  const { from, to } = state.selection;
  
  if (from === to) return; // 无选区不操作
  
  let tr = state.tr;
  
  // 1. 移除目标选区上所有现有的 inline marks（完全覆盖）
  for (const markType of Object.values(state.schema.marks)) {
    // 跳过 link——不主动删除目标文本的链接
    if (markType.name === 'link') continue;
    tr = tr.removeMark(from, to, markType);
  }
  
  // 2. 添加采集到的 marks
  for (const mark of marks) {
    if (mark.type.name === 'link') continue; // 再次跳过 link
    tr = tr.addMark(from, to, mark);
  }
  
  view.dispatch(tr);
}
```

### Step 3: 状态管理

```ts
interface FormatPainterState {
  active: boolean;        // 是否激活
  locked: boolean;        // 锁定模式（双击触发）
  marks: Mark[] | null;   // 采集到的格式
}
```

模块级变量（每个 editor 实例独立）：

```ts
const painterStateMap = new WeakMap<Zotero.EditorInstance, FormatPainterState>();
```

### Step 4: 事件监听

格式刷激活后，需要监听编辑区域的 `mouseup` 事件来检测用户选中了新文本：

```ts
function onMouseUp(view: EditorView, editor: Zotero.EditorInstance): void {
  const state = painterStateMap.get(editor);
  if (!state?.active || !state.marks) return;
  
  const { from, to } = view.state.selection;
  if (from === to) return; // 用户只是点击，没有选区
  
  // 应用格式
  applyFormat(view, state.marks);
  
  // 单击模式：应用后退出
  if (!state.locked) {
    deactivateFormatPainter(editor);
  }
}
```

退出机制：

```ts
function onKeyDown(e: KeyboardEvent, editor: Zotero.EditorInstance): void {
  if (e.key === 'Escape') {
    deactivateFormatPainter(editor);
  }
}
```

### Step 5: UI 注入（同 colorPicker 模式）

使用 MutationObserver 监听 `.text-dropdown .popup` 出现：

```ts
const observer = new IframeMutationObserver((mutations: MutationRecord[]) => {
  for (const mutation of mutations) {
    if (mutation.type === "childList") {
      mutation.addedNodes.forEach((node) => {
        if (IframeElement && node instanceof IframeElement) {
          const el = node as Element;
          const popups = el.matches(".text-dropdown .popup")
            ? [el]
            : Array.from(el.querySelectorAll(".text-dropdown .popup"));
          popups.forEach((popup) => {
            injectFormatPainterButton(popup as HTMLElement, editor, doc);
          });
        }
      });
    }
  }
});

observer.observe(toolbar, { childList: true, subtree: true });
```

按钮注入：

```ts
function injectFormatPainterButton(
  popup: HTMLElement, 
  editor: Zotero.EditorInstance, 
  doc: Document
): void {
  if (popup.querySelector(".researchopia-format-painter-btn")) return;
  
  const separator = doc.createElement("div");
  separator.className = "separator";
  separator.style.cssText = "height:1px;background-color:var(--color-panedivider,#e0e0e0);margin:4px 9px;";
  
  const btn = doc.createElement("button");
  btn.className = "option researchopia-format-painter-btn";
  btn.setAttribute("role", "menuitem");
  btn.innerHTML = `
    <div class="icon">
      <svg width="16" height="16" viewBox="0 0 16 16" fill="none">
        <path d="M10.5 1H3.5C3.22 1 3 1.22 3 1.5V5.5C3 5.78 3.22 6 3.5 6H4V7H5V6H9V7H10V6H10.5C10.78 6 11 5.78 11 5.5V1.5C11 1.22 10.78 1 10.5 1ZM10 5H4V2H10V5Z" fill="currentColor"/>
        <path d="M8 7H6V9L4 15H6.5L8 9V7Z" fill="currentColor"/>
      </svg>
    </div>
    <div class="name">Format Painter</div>
  `;
  
  // 单击 / 双击 处理
  let clickTimer: number | null = null;
  
  btn.addEventListener("mousedown", (e) => {
    e.preventDefault(); // 防止下拉菜单关闭和失去编辑器聚焦
  });
  
  btn.addEventListener("click", (e) => {
    e.preventDefault();
    e.stopPropagation();
    
    if (clickTimer) {
      // 双击 → 锁定模式
      clearTimeout(clickTimer);
      clickTimer = null;
      activateFormatPainter(editor, true /* locked */);
    } else {
      // 延迟判断是否为单击
      clickTimer = setTimeout(() => {
        clickTimer = null;
        activateFormatPainter(editor, false /* one-shot */);
      }, 250);
    }
    
    // 关闭下拉菜单
    closeDropdown(popup.closest(".text-dropdown") as HTMLElement);
  });
  
  popup.appendChild(separator);
  popup.appendChild(btn);
}
```

### Step 6: 激活状态指示器

下拉菜单关闭后，用户需要知道格式刷是否仍然激活。在 toolbar 上添加一个临时指示器：

```ts
function showToolbarIndicator(editor: Zotero.EditorInstance, locked: boolean): void {
  const doc = editor._iframeWindow?.document;
  if (!doc) return;
  
  // 在 toolbar 的 .text-dropdown 按钮上添加激活样式
  const textDropdown = doc.querySelector('.text-dropdown');
  if (textDropdown) {
    textDropdown.classList.add('researchopia-fp-active');
    if (locked) textDropdown.classList.add('researchopia-fp-locked');
  }
  
  // 编辑区域添加特殊光标
  const prosemirror = doc.querySelector('.ProseMirror');
  if (prosemirror) {
    (prosemirror as HTMLElement).classList.add('researchopia-fp-cursor');
  }
}
```

### Step 7: 清理

```ts
export function removeFormatPainter(editor: Zotero.EditorInstance): void {
  deactivateFormatPainter(editor);
  // 断开 MutationObserver
  const observer = fpObservers.get(editor);
  if (observer) {
    observer.disconnect();
    fpObservers.delete(editor);
  }
  // 移除注入的样式
  const doc = editor._iframeWindow?.document;
  if (doc) {
    doc.getElementById("researchopia-format-painter-styles")?.remove();
  }
}
```

## Edge Cases

| 场景 | 处理 |
|------|------|
| 源文本包含 link mark | 过滤掉，不复制链接 |
| 目标文本已有格式 | 先清除所有 marks，再应用采集到的（完全覆盖） |
| 源文本为纯文本（无 marks） | marks = []，效果等同于「清除格式」 |
| 选区跨多个 block 节点 | 仅处理 inline marks，block 格式（heading/list）不变 |
| 编辑器切换 note | 清除格式刷状态 |
| 窗口关闭 | removeFormatPainter 清理 |
